# HTML/JS Development Rules (General Standard)

## 1. アーキテクチャとモジュール化 (Architecture)
- **関心の分離 (Separation of Concerns)**: HTML（構造）、CSS（スタイル）、JavaScript（ロジック）は原則として別ファイルに分割すること。インラインスクリプトやインラインスタイルは最小限に留める。
- **ES6+モジュールの活用**: 大規模なスクリプトは単一ファイルに詰め込まず、`export` / `import` を用いて機能ごとにモジュール化（分割）し、保守性を高めること。

## 2. パフォーマンスと非同期処理 (Performance & Async)
- **メインスレッドのブロック回避**: 重い計算処理でUIをフリーズ（画面固まり）させないこと。
- **モダンな非同期通信**: コールバック地獄を避け、ネットワークリクエストや非同期処理には `fetch` API と `async/await` を標準として使用すること。
- **イベントの最適化**: `scroll` や `resize`、連続した `keyup` など頻発するDOMイベントには、必ずデバウンス（Debounce）またはスロットリング（Throttle）処理を噛ませ、ブラウザの負荷を抑えること。

## 3. DOM操作と状態管理 (DOM & State)
- **DOMアクセスの最小化**: DOMの検索（`document.querySelector` など）はコストが高いため、ループ内で何度も行わず、一度変数にキャッシュして再利用すること。
- **状態（State）の分離**: UIの表示状態（DOM）とデータ（State）を密結合させない。データが変更されたらUIを再レンダリングする「データ主導」の設計を意識すること。

## 4. UI/UXとアクセシビリティ (UI/UX & a11y)
- **セマンティックHTML**: `<div>` や `<span>` の乱用を避け、`<header>`, `<main>`, `<nav>`, `<button>` などの意味論的に正しいタグを使用すること。
- **レスポンシブ設計**: CSSはモバイルファーストを意識し、メディアクエリ（`@media`）を用いてPC・スマホ・タブレットのあらゆる画面サイズに適切に対応させること。

## 5. セキュリティとエラーハンドリング (Security & Error Handling)
- **XSS（クロスサイトスクリプティング）対策の徹底**: ユーザーの入力値や外部データをDOMに挿入する際は、必ずサニタイズ（無害化）処理を行うか、`textContent` を使用してHTMLエスケープを徹底すること。**`innerHTML` の安易な使用は絶対厳禁とする。**
- **堅牢な例外処理**: `try...catch` ブロックを適切に配置し、エラー発生時はコンソールにログ（`console.error`）を出力するだけでなく、ユーザーに分かりやすい形でUIにフィードバック（エラーメッセージ）を表示すること。

## 6. バージョン管理とGit運用 (Version Control)
- **コミットメッセージ**: 日本語で簡潔に（例: `fix: バグ修正内容`, `feat: 新機能内容`, `refac: リファクタリング内容`）記述すること。
- **無視ファイルの徹底**: `node_modules` フォルダ、`.env`（環境変数）、その他ビルド生成物など、Git管理に不要かつ機密性の高いファイルは必ず `.gitignore` に含めること。
