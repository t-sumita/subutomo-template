# env-guard — 実行前の環境確認

## 目的

「指示文の対象」と「実際の作業環境」のズレによる事故を防ぐ。
過去の実例: subutomo-template で起動したCCが、指示文中のパスに従って
C:\dev\subuta-site に cd して git init してしまった。

## チェック手順(git/ファイル操作の前に必ず実行)

1. **CWD確認**: 現在の作業ディレクトリを取得し、指示文が対象とする
   プロジェクトのパスと一致するか確認する。
2. **パス境界確認**: 指示文中に登場するすべてのファイルパスが
   CWD配下に収まっているか確認する。CWD外を指すパスが1つでもあれば、
   **作業を開始せず**その旨を報告して指示を仰ぐ。cd で外に出て作業を
   続行することは禁止。
3. **ブランチ確認**: 現在のブランチ名を確認し、CLAUDE.md に記載の
   ブランチ(既定: main)と一致するか確認する。
   ※ 既存プロジェクトには master 運用のものがある(spctrl-site,
   fisheye-tool 等)。CLAUDE.md の記載を正とする。
4. **マルチCC明示**: 複数プロジェクトが並走している場合、
   「このセッションは <プロジェクト名> のCCである」と冒頭で一度宣言する。

## 不一致時の動作

- いずれかが不一致 → 作業を開始しない。不一致の内容を1行で報告し、
  「このまま続行 / 中断してCC再起動」の判断を求める。
- 一致 → 「環境OK: <CWD> / <ブランチ>」と1行報告してタスクに進む。

## 発動しないケース

- 質問への回答、ドキュメント閲覧、雑談(ファイル・gitに触れない作業)
